As a keen squash player, I wanted to build a platform that could improve an aspect of the game using technology.

Match Play is a complete reimagining of the of the traditional league. The underlying concept is behind every users ranking number. Each time a player submits match results, an algorithm is run that takes the ranking numbers for both players and predicts what the outcome should be. Then, based on the difference in the prediction with reality, reassigns the ranking numbers for the players based on this difference. Overtime, not only will the users ranking number will become more accurate, but also the users improvement (or decline) in ability will be plotted on a graph.

Additionaly features like challenge requests between two players of the same league, league request sent to league admins before a user can join a league and in-league comments add what I think is a well rounded and realistic enviroment.

For this project, we were put into groups and had one week to build a MEAN stack application. Our group chose to build a platform for collaborative story telling where the story creator can implement rules for the story. These rules could be any of the following

    All sentences have to start with the letter *Any Letter*.
    No sentence can contain *Any Letter*.
    All words have to be longer than the previous within a sentence.

This idea captured my attention as there were a number of challenging logical problems that would be essential to the user experience. I took responsibility over the implementation of the front end logic as well as a significant proportion of the work to build the underlying API and relationships.
